Authentication
                

API Endpoint
https://api.acessorias.com

# Curl
curl --location --request GET 'https://api.acessorias.com'
--header 'Authorization: Bearer secret_key'
                

            
Utilizamos o padrão Bearer Authentication passado no Header das requisições.

Para utilizar a API, é necessário gerar o seu Token através do botão da "engrenagem", no canto superior direito do Sistema Acessórias e clicar na opção "API Token".

Rate-Limit
Com o intuito de preservar os recursos e disponibilidade dos nossos sistemas, a API conta com a limitação de requisições.

O limite é de 100 requisições por minuto, usando como estratégia de limitação o algoritmo 'Sliding Window', que libera novos slots de requisições conforme antigos slots são liberados.

    'Authorization: Bearer secret_key'

Detalhes da requisição
Campo	Tipo	Descrição
secret_key	String	API Token
cnpj	String	Identificador da empresa, que pode ser CNPJ, CPF ou CAEPF. Caso já exista algum registro com o identificador enviado, o mesmo será atualizado.
nome	String	Razão social da empresa
fantasia	String	Nome fantasia da empresa
id	Integer	(opcional) ID da empresa
getid	Char	(opcional) [S/N] Valor padrão deste campo é: 'N'. Caso o ID enviado já esteja sendo utilizado por outra empresa, o registro será ignorado, a menos que este campo seja enviado com valor: 'S', neste caso o sistema irá buscar o próximo ID disponível.
dtcadastro	Date	(opcional) [Y-m-d] Data de cadastro da empresa.
dtclidesde	Date	(opcional) [Y-m-d] Data que o cliente entrou no escritório.
dtcliate	Date	(opcional) [Y-m-d] Data que o cliente saiu do escritório.
regime	Integer	(opcional) [Só será considerado em novos registros] ID do regime tributário da empresa, sendo:
0	Indefinido
1	Simples nacional com inscrição estadual
2	Simples nacional sem inscrição estadual
3	Lucro presumido com inscrição estadual - industria/comércio
4	Lucro presumido sem inscrição estadual - serviço

6	MEI - Micro Empreendedor Individual
7	Domésticas/Caseiro - e-Social
8	Produtor Rural
9	Pessoa Física
10	Imune/Isenta
dtabertura	Date	(opcional) [Y-m-d] Data de abertura da empresa.
inscmunicipal	String	(opcional) Inscrição municipal da empresa.
dtinscmunicipal	Date	(opcional) [Y-m-d] Data da inscrição municipal da empresa.
nire	String	(opcional) Nire da empresa.
apelido	String	(opcional) Apelido da empresa para o e-Contínuo.
endlogradouro	String	(opcional) Endereço.
endnumero	String	(opcional) Número do endereço.
endcomplemento	String	(opcional) Complemento do endereço.
cep	String	(opcional) CEP.
bairro	String	(opcional) Bairro.
cidade	String	(opcional) Cidade.
uf	String	(opcional) Estado.
codibge	String	(opcional) Código do IBGE.
website	String	(opcional) Website da empresa.
fone	String	(opcional) Telefone da empresa.
honorario	Float	(opcional) Valor de honorário da empresa. Observação: Milhares sem separação e decimais separados com ponto. Exemplo: 1580.98
ativa	String	(opcional) [S/N] Padrão deste campo: 'S'. Empresa está ativa?.

Pagina	Query Param	Número da página para busca de registros. Caso não seja informado retornará os registros da página 1. Dica: Só são listados 50 registros por página, para listar todos os dados, liste as páginas seguintes até vir uma lista vazia.
config	Query Param	(opcional) Envie esse parâmetro para trazer a ID da entrega, Tipo do prazo (O = Obrigação ou T = Tarefa), ID da obrigação ou tarefa, ID do Dpto e Nome do Dpto.
Caso enviar a ID da entrega nesse parâmetro, apenas os dados da entrega específica será enviada no retorno.

bStatus	Query Param	(opcional) Informe a letra correspondente para filtrar boletos pelos seguintes status:
  P = Pendentes
  Q = Quitados
  C = Cancelados
bID	Query Param	(opcional) Informe a ID do boleto desejado para buscar somente o registro específico
Pagina	Query Param	Número da página para busca de registros. Caso não seja informado retornará os registros da página 1. Dica: Só são listados 20 registros por página, para listar todos os dados, liste as páginas seguintes até vir uma lista vazia.

  X = E(X)cluídos
Obs: Processos excluídos não são listados por padrão e são apenas exibidos quando buscados separadamente.
ProcNome	Query Param	(opcional) Informe o nome ou título do processo para buscar os registros específicos
ProcInicioIni	Query Param	(opcional) Data inicial do processo. Utilizar no padrão USA (YYYY-MM-DD)
ProcInicioFim	Query Param	(opcional) Data final do processo. Utilizar no padrão USA (YYYY-MM-DD)
ProcConclusaoIni	Query Param	(opcional) Data inicial de conclusão do processo. Utilizar no padrão USA (YYYY-MM-DD)
ProcConclusaoFim	Query Param	(opcional) Data final de conclusão do processo. Utilizar no padrão USA (YYYY-MM-DD)
Pagina	Query Param	Número da página para busca de registros. Caso não seja informado retornará os registros da página 1. Dica: Só são listados 20 registros por página, para listar todos os dados, liste as páginas seguintes até vir uma lista vazia.
DtLastDH	Query Param	Opção para listar somente os processos que tiveram alterações após uma data/hora específica. Data no padrão USA (YYYY-MM-DD HH:MM:SS). Exemplo: '2025-11-05 09:57:01'
